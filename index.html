About

Nodewars is a competitive programming game that blends elements of spacial domination and real-time strategy. Set in the not-to-distant future, rival teams of hackers compete to control as many machines as possible.

In this dark future, the only money that still holds any value is a digital currency known as CoinCoin. Teams use the networks they control to mine as much CoinCoin as possible. The more area a team controls, the more CoinCoin they can generate.

Interface

Let's start by getting familiar with the interface. If you're not familiar with programming environments, Nodewars' interface can seem a little overwhelming at first, so let's cover each window and what it does.

(adjusting layout)

Main Window - Contains other windows, Layout menu can be used to save/recall window layouts

Terminal - How you interact with the world, type help any time you get lost

Map - Show who controls what and where you're connected. Useful in planning your next move

Challenge Details - Once you're connected to a machine the programming challenge you have to solve will be described here

Ace Editor - This is where you write your code. You can use the terminal or menu to compile but shortcuts are recommended

Stdin - When you want to test and idea you can use this to feed your own input into your program

Compiler Output - Any compile errors your code generates will appear here

Score - This shows how much CoinCoin each team has. Whoever reaches the top first, wins!

Test Results - This shows you how well you're code did at solving the current challenge

Gameplay

Joining a Game

Before we can do anything in Nodewars, we need to join a game. You can use the ‘ls’ command to see what games are available. Use 'join' to join an ongoing game. If you can't find a game, ‘new’ will start a new game and join it. Once in a game use ‘join’ again to join a team.

Basic Gameplay

There are a number of subtleties in Nodewars but the main gameplay is fairly straightforward. Generally a player will connect (con) to a node, select a machine they'd like to attach (at) to, and begin trying to solve whatever programming challenge is present at that machine.

Rules of Connection
(connection routing)

Each team has a Point of Entry (POE). This is point through which all members of that team access the map. Players can connect to any node adjacent to a node where they control a (powered) machine. A machine is powered as long as it is part of an unbroken chain of machines connecting back to the teams POE. It's simpler than it sounds, take a look at this example.

(cut connection)
So what happens when connections are interrupted?

When a team loses all of the machines it controls in a node, it is no longer able to route connections through that node. That means that any players that were connected somewhere on the map via that node get disconnected.

Another side effect is that any machines that are cannot route to their team's POE lose power. This causes them to stop producing CoinCoin and whatever feature they may have installed loses effect.

Unpowered machines also have their strength reduced to 1 (until power is restored). This reduction makes them extremely vulnerable to takeover.

Mining CoinCoin

CoinCoin (CC) is what separates winners from losers in the world of Nodewars. Each team is trying to reach the CC goal first. Each machine the team controls is able to generate CC for the team and bring them closer to victory, although sometimes it is advantageous to forgo the quick coin in favor of a strategic advantage. We’ll get more into machine types in the Node section below. but for now just note that ring machines are the only one’s capable of mining CC and if a team controls every ring machine in a node and all of those machine’s are mining, production doubles, which can be pretty significant.

Two important details about how CC is generated:
  • A machine must have power in order to function, this means an uninterrupted connection to it's team's POE
  • If all the ring machines in a node are owned by the same team and all are mining, CC generation in that node doubles

Anatomy of a Node

Each node is essentially a mini-network of machines. Each node contains exactly one gateway machine (represented by the center circle) and one or more ring machines (represented by the outer circle/segments)

The two type of machines differ only in what you can do with them once you control them. Ring machines are the only machines that are able to generate CC for your team, and ultimately win you the game, so while they can also have special features installed in them, player’s must think carefully about whether it’s worth sacrificing the coin. Gateway machines never generate CC, although some features can amplify the power of mining ring machines.

Machines and Challenges

 Attaching

In order to take control of a machine, you must complete the programming challenge contained therein. When you connect to a node you will be presented with a list of machine addresses. If for some reason you don’t see it or you want it printed again, just type ‘ls’. To attach to a particular machine type ‘at [address]’. Once you connect to a machine challenge details will be loaded automatically into your code environment.

Test/Make

To test your code (using the custom stdin box for input) run the ‘test’ command in the terminal. When you are satisfied your code works, use the ‘make’ command to attempt to gain control of the machine. When you submit your code it will be checked against a series of tests. A machine has one point of strength for each test it passes. When attempting to control a neutral machine you only need to pass a single test to succeed.

There are a couple of important things to note about the make command; When using make on a ring machine, it will default to creating a mining machine, if you while like to install encryption instead, you must use ‘make encrypt’; When using make on a gateway, you must provide a feature type.

Reset

Dealing with neutral machines is a little different than deal with machines already controlled by an opponent. While a neutral machine requires only one successful test pass, when taking over an enemy machine, your solution must pass at least as many tests as theirs did. If your solution passes more tests, than the make command will steal control of the machine. If, however, your solution ties your opponent’s, make will not work and you will have to reset the machine. Resetting a machine reverts it to neutral status and installs a brand new challenge.

Features

Features allow a machine to have special effects. Some features can only be installed in certain types of machines. If a machine loses power, it's feature will no longer operate.

Mining

Mining is the default for ring machines. Generates CC for the team. The more remote the node is, the more CC is generated. If all ring machines in a node are mining for the same team, CC production doubles.

Overclock

Overclocking pushes the node’s ring machines to produce more coin, double production for each friendly ring machine. Be careful though, overclocking can occasionally cause a machine to overheat and reset.

Encryption

Encryption hampers your opponents ability to attack machines in this node. Opponents encrypted machines must be reset before an other machines in the node can be attached to.

Firewall

A firewall hampers the opponents ability to route connections through the node. Normally if a team controls at least one machine in a node, that team can route unlimited connections through the node. If a firewall is installed a team can only route one connection for each machine they control.
